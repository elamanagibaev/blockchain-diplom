# Многоэтапное согласование и финальная регистрация

## Определения этапов

Этапы задаются в таблице `approval_stage_definitions` (сид по умолчанию создаётся при первом обращении к сервису согласования).

| Порядок | Код | Название | Кто может подтверждать / отклонять |
|---------|-----|----------|-------------------------------------|
| 1 | `DEPARTMENT_REVIEW` | Кафедра | `department`, `admin` |
| 2 | `DEAN_REGISTRAR_REVIEW` | Деканат / регистратор | `dean`, `registrar`, `admin` |

`admin` может действовать на любом этапе.

## Логика на документе

- При подаче заявки (`UNDER_REVIEW`) для документа очищаются предыдущие записи `approval_actions` и этапы проходятся заново.
- **Текущий этап** — первый этап, по которому ещё нет действия `APPROVE`.
- **Подтверждение этапа** — запись в `approval_actions` с `action = APPROVE` и привязкой к `stage_definition_id`.
- **Отклонение** — запись `REJECT`, статус документа → `REJECTED`.
- Когда по всем этапам есть `APPROVE`, статус документа → **`APPROVED`**. В журнал пишется `APPROVAL_COMPLETED`.

## Финальная регистрация (on-chain)

Это **отдельный шаг** после внутреннего `APPROVED`:

- Выполняется администратором (`POST /admin/documents/{id}/approve`).
- Вызывается `BlockchainService.register_on_chain`: транзакция в смарт-контракт, статус → `REGISTERED_ON_CHAIN`.
- Требуется настроенный провайдер Web3, адрес контракта и ключ владельца контракта (см. переменные окружения). Если блокчейн не настроен, API вернёт 503.

Итого: **согласование** = внутренние этапы до `APPROVED`; **финальная регистрация** = одна административная операция в сети.

## API (ориентиры)

| Назначение | Метод |
|------------|--------|
| Подача на согласование | `POST /files/{id}/submit-for-registration` |
| Состояние этапов | `GET /approvals/documents/{id}/stages` |
| Подтвердить этап | `POST /approvals/documents/{id}/approve` |
| Отклонить этап | `POST /approvals/documents/{id}/reject` |
| Очередь по коду этапа | `GET /approvals/pending?stage_code=...` |
| Очередь финальной регистрации | `GET /admin/documents/pending` |
| Регистрация в сети | `POST /admin/documents/{id}/approve` |
