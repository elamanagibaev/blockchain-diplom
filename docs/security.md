## Механизмы безопасности (MVP)

### Аутентификация и авторизация

- **JWT** токены (`/api/auth/login`), срок жизни настраивается через `ACCESS_TOKEN_EXPIRE_MINUTES`.
- **Хэширование паролей**: bcrypt (`passlib[bcrypt]`).
- **RBAC**: роли `user` и `admin` (ограничения проверяются на backend).

### Безопасная загрузка файлов

- **Лимит размера** файла (10MB) — защита от DoS по памяти/диску.
- **Проверка MIME-типа** (разрешены `application/pdf`, `image/*`, `text/*`).
- **Безопасное имя файла**: очищается от опасных символов и traversal.

### Разделение on-chain / off-chain

- На блокчейне — только контрольные данные и история.
- В PostgreSQL + storage — метаданные и бинарный файл.

### Управление секретами

- Секрет JWT и ключи для blockchain берутся из окружения (`.env`), не захардкожены в коде.
- В docker-compose ключ для Hardhat указан как **dev-only** (для демонстрации).
