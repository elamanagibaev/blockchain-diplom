# Сценарий демонстрации на защите

Краткий пошаговый сценарий для демонстрации платформы BlockProof на защите диплома.

## Подготовка

1. Запустить проект: `docker compose up --build -d`
2. Создать демо-данные: `make seed` или `docker compose run --rm backend python -m app.scripts.seed_demo`
3. Открыть браузер: http://localhost:5173

## Демо-доступ (после seed)

- **admin@example.com** / admin — администратор
- **doctor@clinic.ru** / doctor123 — врач (есть демо-документы)
- **patient@example.com** / patient123 — пациент

---

## Сценарий 1: Регистрация и вход

1. Перейти на страницу «Регистрация» → создать нового пользователя
2. Показать валидацию пароля (цифра, заглавная, строчная, 8+ символов)
3. Успешная регистрация → уведомление «Аккаунт создан» → редирект на вход
4. Войти с созданным логином

---

## Сценарий 2: Dashboard и метрики

1. Войти как **doctor@clinic.ru** / doctor123
2. Показать главную страницу:
   - Hero-блок с приветствием
   - Метрики: всего документов, в блокчейне, верифицировано
   - Блок «Последние действия»
   - Список «Последние документы»
3. Перейти в «Мои документы» — показать таблицу демо-файлов

---

## Сценарий 3: Загрузка документа

1. Перейти в «Загрузка документа»
2. Выбрать PDF или изображение
3. Опционально: указать описание
4. Нажать «Загрузить»
5. Показать результат: статус, хэш SHA-256, ссылка «Открыть детали»

---

## Сценарий 4: Регистрация в блокчейне

1. Открыть детали документа (например, «Результаты_анализа_крови.pdf»)
2. Показать: метаданные, хэш, off-chain/on-chain история
3. Нажать «Зарегистрировать в блокчейне» (если ещё не зарегистрирован)
4. Показать tx_hash и статус «В блокчейне»

---

## Сценарий 5: Верификация документа

1. Перейти на страницу «Проверка подлинности»
2. Загрузить **тот же файл**, что был загружен ранее
3. Показать результат: «Документ верифицирован», сравнение хэшей, blockchain proof
4. Опционально: «Экспорт» и «Печать» для PDF-отчёта

---

## Сценарий 6: Админ-панель

1. Войти как **admin@example.com** / admin
2. Перейти в «Админ-панель»
3. Показать: Health-статус, проект, версия
4. Показать таблицу пользователей, смену роли (user ↔ admin)

---

## Тезисы для защиты

- **Off-chain**: файл хранится в локальном хранилище (или MinIO)
- **On-chain**: в блокчейне только SHA-256, timestamp, owner, metadataURI
- **Верификация**: сравнение хэшей + проверка записи в смарт-контракте
- **Безопасность**: rate limiting, валидация паролей, CORS, JWT
