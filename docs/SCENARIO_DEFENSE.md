# Сценарий демонстрации на защите

Краткий пошаговый сценарий для демонстрации платформы BlockProof на защите диплома.

## Подготовка

1. Запустить проект: `docker compose up --build -d`
2. Создать демо-данные: `make seed` или `docker compose run --rm backend python -m app.scripts.seed_demo`
3. Открыть браузер: http://localhost:5173

## Демо-доступ (после seed)

- **admin@example.com** / admin — администратор
- **patentee@ip.ru** / patentee123 — патентообладатель (есть демо-документы)
- **inventor@example.com** / inventor123 — дополнительный пользователь

---

## Сценарий 1: Регистрация и вход

1. Перейти на страницу «Регистрация» → создать нового пользователя
2. Показать валидацию пароля (цифра, заглавная, строчная, 8+ символов)
3. Успешная регистрация → уведомление «Аккаунт создан» → редирект на вход
4. Войти с созданным логином

---

## Сценарий 2: Панель и метрики

1. Войти как **patentee@ip.ru** / patentee123
2. Показать главную страницу:
   - приветствие и краткое описание
   - метрики: всего документов, в блокчейне, на рассмотрении
   - блок «Последние документы»
   - при наличии — «События в блокчейне»
3. Перейти в «Мои патенты» — показать таблицу демо-файлов

---

## Сценарий 3: Загрузка документа

1. Перейти в «Загрузка»
2. Выбрать PDF или изображение
3. Опционально: описание (номер заявки, объект ИС)
4. Нажать «Загрузить и зафиксировать хэш»
5. Показать результат: статус, SHA-256, ссылка «Открыть карточку»

---

## Сценарий 4: Регистрация в блокчейне

1. Открыть карточку документа (например, из демо-набора)
2. Показать метаданные, хэш, блок «Блокчейн»
3. Нажать «Подать на регистрацию» (если ещё не в сети)
4. Войти как администратор и одобрить заявку в админ-панели
5. Показать tx hash и статус «В блокчейне»

---

## Сценарий 5: Верификация

1. Перейти на «Верификация»
2. Загрузить **тот же файл**, что был зарегистрирован ранее, или ввести SHA-256
3. Показать результат: статус целостности, сравнение хэшей, данные on-chain
4. Открыть сертификат по ссылке при необходимости

---

## Сценарий 6: Админ-панель

1. Войти как **admin@example.com** / admin
2. Перейти в «Админ»
3. Показать: Health-статус, проект, версия
4. Показать таблицу пользователей, смену роли (user ↔ admin)

---

## Тезисы для защиты

- **Off-chain**: файл в локальном хранилище или MinIO
- **On-chain**: в блокчейне SHA-256, время, кошелёк правообладателя, metadataURI
- **Верификация**: сравнение хэшей и сверка со смарт-контрактом
- **Безопасность**: rate limiting, валидация паролей, CORS, JWT
