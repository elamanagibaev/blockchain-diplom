# Верификация подлинности (verify flow)

Подлинность **подтверждается только** если документ в статусе `REGISTERED_ON_CHAIN` и у него заполнен `blockchain_tx_hash` (запись в реестре смарт-контракта согласована с хэшем в БД).

## Проверка по файлу

1. Клиент отправляет файл на `POST /api/verify/file` (multipart).
2. Сервер вычисляет SHA-256 содержимого.
3. Ищется `DigitalObject` с таким `sha256_hash`.
4. Результат:
   - **NOT_FOUND** — хэш ни у кого не зарегистрирован.
   - **INVALID** — объект найден, но нет полноценной on-chain регистрации (или другой статус).
   - **VALID** — объект найден и `REGISTERED_ON_CHAIN` с транзакцией.

## Проверка по хэшу

`GET /api/verify/hash/{sha256}` — та же логика поиска по строке хэша (64 hex-символа). Некорректный формат → **INVALID_HASH**.

## Публичная страница по ID документа

`GET /api/verify/{document_id}` — без авторизации; отдаётся краткая справка (статус, владелец, фрагмент хэша, флаг `is_authentic`).

- **is_authentic = true** только при `REGISTERED_ON_CHAIN` и наличии `blockchain_tx_hash`.
- Подлинность **файла по байтам** пользователь подтверждает на странице проверки с загрузкой файла или вводом полного SHA-256; публичная карточка по ID не заменяет сравнение хэша.

Фронтенд: маршрут `/verify/doc/:docId` (публичный), полная проверка — `/verify`.

## QR

На карточке документа формируется ссылка вида `{origin}/verify/doc/{id}` — это вход на публичную справку; для демонстрации «подлинности файла» используйте загрузку файла на `/verify`.

## Логирование

Запросы верификации могут отражаться в `verification_logs` и в журнале `document_events` (`VERIFY_*`).

## Где что лежит

| Место | Роль |
|-------|------|
| **PostgreSQL** | Хэш документа, статус, связь с владельцем, логи проверок. |
| **MinIO / файловое хранилище** | Исходный файл по `storage_key` (для скачивания владельцем; verify пересчитывает хэш с загрузки). |
| **Blockchain** | Неизменяемая фиксация хэша в контракте; валидность «VALID» согласована с этой записью. |

Общий жизненный цикл: [document-lifecycle.md](./document-lifecycle.md).
